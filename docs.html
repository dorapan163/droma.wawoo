Getting Started

Droma is a mobile console for running large language models entirely on your device. No cloud account is required after setup.

After installing the app, complete the first-run wizard to pick a default model tier and enable on-device encryption for stored weights.

Open the Console tab to start your first conversation. All inference happens locally using your phone's CPU, GPU, or NPU.

Installation

Download Droma from Google Play or the App Store, or scan the QR code on the download page of our website.

Grant storage permission when prompted so the app can import and cache model files. Optional microphone access is only needed if you use voice input.

Keep at least 6 GB of free RAM and sufficient storage for your chosen models (typically 4–12 GB per 7B-class model depending on quantization).

Model Hub

Open Model Hub from the bottom navigation to browse the curated catalog or import your own GGUF, ONNX, or Droma Pack (.drp) files.

Tap a model card to see size, quantization, and estimated tokens per second on your device tier. Use one-tap quantization to match your hardware.

Downloaded weights are stored in the app sandbox and encrypted at rest with AES-256-GCM. Delete unused models from Model Hub to free space.

Chat

The Chat screen supports multiple parallel sessions. Each session keeps its own context window and history on device.

Long-press a message to copy, regenerate, or branch the conversation. Swipe left on a session in the drawer to archive or delete it.

Streaming responses appear token-by-token. If generation feels slow, try a smaller quantization (e.g. Q4_K_M) or reduce context length in settings.

Inference Settings

From Settings → Inference, adjust temperature, top-p, top-k, repeat penalty, and max context length before each session or set defaults globally.

Enable NPU / GPU acceleration when your chipset supports it. The app falls back to CPU (NEON / AVX2) automatically if hardware paths are unavailable.

Advanced users can export a diagnostic log (no message content) to help support tune backend selection.

Bridge API

Droma Bridge exposes OpenAI-compatible HTTP and WebSocket endpoints on your local network while the app is in the foreground.

Use the API tab to copy the base URL and API key. Third-party clients can list models, create chat completions, and stream tokens against your loaded local model.

Troubleshooting

Model fails to load: verify free storage, re-download the weight file, and ensure the format is supported (GGUF recommended).

App closes under load: close background apps, lower context length, or switch to a smaller model. Thermal throttling may temporarily reduce speed.

No network features work offline by design. Bridge API is only available when Droma is open and the local server toggle is enabled.

FAQ

Is my data sent to the cloud?
No. Chat history and model weights stay on your device unless you explicitly export or use Bridge API on your local network.
Which models are supported?
GGUF is the primary format. ONNX Mobile and custom Droma Packs are supported via Model Hub import pipelines.
How do I update the app?
Updates are delivered through your platform store like any other app. Model files are managed separately inside Model Hub.
